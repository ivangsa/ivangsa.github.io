This talk introduces the API-First methodology applied to event-driven systems using AsyncAPI as the contract definition format.

Topics include:

- What API-First means for event-driven architectures
- Designing AsyncAPI documents before writing any code
- Generating boilerplate producers and consumers from AsyncAPI specs
- Keeping teams aligned through shared AsyncAPI contracts
